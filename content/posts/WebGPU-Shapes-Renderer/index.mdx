  <h1 class="content-heading-1">WebGPU Shapes Renderer: Exploring Modern Web Graphics</h1>
  <p class="content-paragraph">
    WebGPU is an emerging API that promises to bring high-performance graphics to the web, rivaling native applications. In this post, I detail my project, the WebGPU Shapes Renderer, which demonstrates 2D shape rendering using WebGPU and Canvas, and share the insights gained from diving into this cutting-edge technology.
  </p>
  
  <h2 class="content-heading-2">What is WebGPU?</h2>
  <p class="content-paragraph">
    WebGPU is a successor to WebGL, designed to provide low-level access to GPU hardware for web applications. It enables developers to perform complex computations and render high-quality graphics directly in the browser, leveraging modern GPU capabilities.
  </p>
  
  <h2 class="content-heading-2">Project Overview</h2>
  <p class="content-paragraph">
    The WebGPU Shapes Renderer is a demo application that renders 2D shapes using WebGPU and Canvas. It showcases basic geometry rendering and includes an OBJ parser to potentially extend to 3D models. Note that WebGPU currently requires Chrome to run.
  </p>

  <h2 class="content-heading-2">Key Features</h2>
  <ul class="content-list">
    <li class="content-list-item">Rendering of basic 2D shapes using WebGPU shaders and buffers.</li>
    <li class="content-list-item">Custom OBJ parser for loading geometry data, with potential for 3D expansion.</li>
    <li class="content-list-item">Interactive canvas display within a browser environment.</li>
  </ul>
  
  <h2 class="content-heading-2">Development Challenges</h2>
  <ul class="content-list">
    <li class="content-list-item"><strong>Learning Curve</strong>: WebGPU is low-level and new, with limited documentation. I tested basic examples repeatedly, breaking shader code to understand GPU pipeline stages.</li>
    <li class="content-list-item"><strong>Buffer Management</strong>: Mapping data to GPU buffers was error-prone. I broke rendering with incorrect mappings, then fixed them by testing smaller datasets and validating each step.</li>
  </ul>
  
  <h2 class="content-heading-2">What I Learned</h2>
  <ul class="content-list">
    <li class="content-list-item">Fundamentals of GPU programming with WebGPU, including shaders, buffers, and pipelines.</li>
    <li class="content-list-item">Debugging graphics code by isolating components and testing incrementally.</li>
    <li class="content-list-item">The potential of web graphics to rival native apps, opening doors to complex visualizations in browsers.</li>
  </ul>
